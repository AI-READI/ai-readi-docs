## Azure Storage Setup Instructions

### Overview

Due to the size of the AI-READI, we require requestors to provide an Azure storage container to receive the processed dataset. Specifically, at the end of the request process you will need to provide:

1. Storage Account Name
1. Blob Container Name
1. Blob SAS Token (with "Write" permission only)

Follow the detailed steps below to create your Azure environment. Please do this before you attempt to request the AI-READI dataset as it can lead to delays in processing your request.

### Create a Microsoft Azure Account

If you don’t already have a Microsoft Account:

- Visit: https://azure.microsoft.com/en-us/free/
- Choose the Free Tier to start; it becomes pay-as-you-go after the trial period.
  A credit card is required for identity verification, but no charges apply unless usage exceeds the free limits.

### Create an Azure Storage Account

Next, create an [Azure Storage Account](https://learn.microsoft.com/en-us/azure/storage/common/storage-account-overview) that contains the container storage. Below is a summarized version of the [full documentation](https://learn.microsoft.com/en-us/azure/storage/common/storage-account-create).

- Log in to the Azure Portal: https://portal.azure.com
- Search for "Storage accounts" in the top search bar and click "+ Create".

  <figure>
    <img src="/img/azure/azure-storage-account-1.png" />
    <figcaption>Locate the Storage account item from the menu in the upper left</figcaption>
  </figure>

- Fill out the form:

  - **Subscription**: Select your plan.
  - **Resource group**: Use an existing one or create a new one.
  - **Storage account name**: Choose a globally unique, lowercase name.
  - **Region**: Choose a region closest to you.
  - **Performance**: Standard (default).
  - **Redundancy**: Locally-redundant storage (LRS) is recommended.

- Click "Review + create", then "Create".
  **Note**: Save your Storage Account Name. You’ll need this when requesting the dataset.

### Create a Blob Storage Container

At this point we now need a place to put the files. As the [Microsoft documentation](https://learn.microsoft.com/en-us/azure/storage/blobs/blob-containers-portal) states "Azure Blob Storage allows you to store large amounts of unstructured object data." This will be where the AI-READI files will be ultimately delivered.

- In the Azure Portal, go to "Storage accounts" and click your storage account.
- In the left sidebar under "Data storage", click "Containers".

  {' '}

  <figure>
    <img src="/img/azure/azure-container-1.png" />

- Click "+ Container".

- Set:
  - Name: Must be lowercase, no spaces.
  - Public access level: Select "Private (no anonymous access)".

**Note**: Save your Container Name for use in the dataset request.

### Generate a SAS Token

(Write Permission Only)

- Inside your container, click the "... More" menu at the top right and select "Generate SAS".
  - **Permissions**: Check only the box for "Write".
  - **Expiry**: Choose a date several weeks or months in the future (e.g., 2 to 3 weeks).
- Click "**Generate SAS token and URL**".
- Copy only the "Blob SAS token" (not the full SAS URL). It will be in the following format:

  _`sp=w&st=2025-06-02T23:28:30Z&se=2025-07-04T07:28:30Z&spr=https&sv=2024-11-04&sr=c sig=KBj2FOV...`_

**Important**: Only submit the SAS token (do not include the full SAS URL).

### Final Checklist

Make sure you have all three of the following:

- Storage Account Name
- Blob Container Name
- Blob SAS Token (Write permission only)

Provide this information via the dataset request form to receive your AI-READI dataset.
